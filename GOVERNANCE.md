# Governance

Kooperativet.ai should be governed in public where possible. The project can start maintainer-led, but decisions that affect community norms, moderation, data handling or contributor rights should be visible and discussable.

## Roles

- Contributors: propose, discuss, design, code, document and test.
- Maintainers: review pull requests, keep the project coherent and protect user trust.
- Moderators: handle community reports and enforce the code of conduct.
- Stewards: lead focused areas such as security, prompt bank, governance, accessibility or inclusion.

One person can hold more than one role, but sensitive actions should be reviewable by others.

## Decision Process

1. Open an issue for product, policy, governance or architecture changes.
2. Label the issue by area and impact.
3. Discuss tradeoffs, risks and alternatives.
4. Maintainers or stewards summarize the decision.
5. Implementation happens through pull requests.
6. Decisions that affect community rules are recorded in documentation.

## Voting

Early voting can be lightweight: reactions, comments and maintainer summaries. As the community grows, the project should add a formal proposal process for high-impact decisions.

High-impact decisions include:

- moderation policy
- data retention
- default privacy settings
- role permissions
- monetization or sponsorship rules
- license changes
- governance changes

## Cooperative Defaults

- Prefer transparent decisions over private coordination.
- Prefer small reversible changes over large opaque rewrites.
- Give newcomers clear paths to contribute.
- Rotate stewardship when possible.
- Make community safety and trust explicit acceptance criteria.
