# Contributing

Kooperativet.ai should be built like the community it wants to host: open, practical, respectful and accountable.

## Ways To Help

- Improve the React prototype, accessibility, responsiveness and interaction details.
- Design data models for auth, profiles, topics, comments, prompt versions, votes, flags and moderation decisions.
- Write tests for community-critical flows such as posting, flagging, voting and settings.
- Improve documentation, onboarding and issue templates.
- Propose governance, moderation or inclusion improvements through issues.

## Development

```bash
npm install
npm run dev
npm run build
```

Before opening a pull request:

- keep changes focused
- run `npm run build`
- describe the user-facing impact
- include screenshots for UI changes
- call out security, privacy or moderation impact when relevant

## Community Product Bar

Changes should support at least one of these outcomes:

- members can learn faster
- members can build together more easily
- moderators can make fair decisions with better context
- newcomers feel safer asking questions
- reusable knowledge becomes easier to find and maintain

## Decision Style

Small code improvements can be handled by normal pull requests. Product, moderation, governance or data-policy changes should start as an issue or RFC so the community can discuss tradeoffs before implementation.
